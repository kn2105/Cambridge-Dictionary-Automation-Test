# 📚 Cambridge Dictionary - Automation & Performance Testing

## 📌 Project Overview
This project focuses on automated functional testing and performance testing for the **Cambridge Online Dictionary** website (`dictionary.cambridge.org`). It covers key user flows such as vocabulary search, UI verification, and system response performance under load.

---

## 🛠️ Tools & Technologies
- **Automation Testing:** Katalon Studio (Groovy/Java-based scripts)
- **Performance Testing:** Apache JMeter
- **Version Control:** Git & GitHub

---

## 🧪 Testing Scope & Highlights

### 1. Manual & Functional Testing
- Designed and executed **30+ test cases** covering critical user journeys.
- Target features: Vocabulary Search, Filter/Translation Options, User Registration/Login, and Profile Management.

### 2. Automation Testing (Katalon Studio)
- Developed automated test scripts for high-frequency search flows and navigation.
- Implemented assertion checks for web elements, page titles, and search result accuracy.
- Built reusable test suites to optimize **Regression Testing** execution time.

### 3. Performance & Load Testing (Apache JMeter)
- Built and executed **14+ test scenarios** to measure application behavior under stress.
- Analyzed key performance metrics: **Response Time, Throughput, Error Rate, and Latency**.

### 4. Defect Management (Jira)
- Tracked defects through the complete **Defect Lifecycle**.
- Logged detailed bug reports including steps to reproduce, expected vs. actual results, logs, and screenshots.
- Executed re-testing to verify bug fixes before closing issues.
